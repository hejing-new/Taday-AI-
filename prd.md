📱 第一部分：C 端（普通用户对话视窗）
这是前端页面，主要目标是给用户提供极其丝滑的提问体验。

核心对话交互 (Chat UI)

流式输出 (Streaming)： 像 ChatGPT 一样打字机效果输出，而不是转圈等半分钟突然蹦出一大段。

Markdown 渲染： 支持完美渲染表格、加粗、代码块（特别是金融财报里的数据表）。

数据溯源 (Citations)： 极其重要！大模型回答完后，必须在底下附带“参考来源”（如：参考《宁德时代2025年报》第15页），并支持点击查看原文片段。这是 RAG 产品的灵魂。

会话管理 (Session Management)

历史会话侧边栏： 用户可以像用 ChatGPT 一样，保留并随时切换历史聊天记录。

一键清空/新建对话： 避免上一个问题的上下文污染下一个问题。

用户反馈机制 (User Feedback)

点赞/点踩 (👍/👎)： 每条回答旁边加个按钮。用户点踩的数据，会自动推送到后台，供你后续重点调优。

🏢 第二部分：B 端（后台运营与系统管理）
这是你提到的重点（后端登录后的界面），主要给业务人员或系统管理员使用。

工作台与数据看板 (Dashboard) —— 你提到的“统计前十”就在这

核心指标总览： 今日提问总数、活跃用户数、Token 消耗量/预估成本。

热词与高频榜： 用户提问最多的 Top 10 问题、Top 10 关键词（支持按日/周/月筛选）。

未命中榜单 (Zero-hit Queries)： 统计那些“大模型回答不上来/向量库里搜不到”的问题。这能指导业务人员接下来该去补充什么文档。

知识库管理中心 (Knowledge Base Management) —— 你提到的“文档审核”就在这

文档上传与解析状态： 支持拖拽上传 PDF/Word 等，展示解析进度（解析中、切片中、入库成功、失败）。

文档状态机（审核流）： * 草稿/解析中 -> 待审核 -> 已发布 (生效) -> 已下线。

业务人员可以在“待审核”状态下，预览系统切分好的段落（Chunks），确认没乱码后再点击“发布入库”。

文档切片 (Chunk) 手动干预： 如果发现财报某一段切错了，管理员可以直接在网页上修改这段文本，并重新向量化。

系统与智能体配置 (Agent Settings)

Prompt 动态配置： 可以在后台直接修改大模型的人设和系统提示词，而不需要改代码、重启服务。

模型与大模型参数配置： 下拉选择使用哪个大模型（比如 Qwen-72B 或 DeepSeek-V3），配置温度值（Temperature）。

用户与权限管理 (RBAC)

角色划分： 超级管理员（管一切）、运营人员（只能看数据和传文档）、普通用户（只能聊天）。

操作日志 (Audit Log)： 记录“谁在什么时间删除了哪份文档”，防止内部员工误操作导致线上库崩溃。

⚙️ 第三部分：非功能性需求与底层架构建议
前后端分离栈：

前端推荐：Vue 3 + Element Plus 或 React + Ant Design（如果是极速开发做 Demo，可以用 Streamlit）。

后端推荐：Python FastAPI（天生支持异步和流式传输，最适合 AI 项目）+ SQLAlchemy。

数据库：PostgreSQL / MySQL (存用户、日志、会话) + ChromaDB (存向量文档)。
